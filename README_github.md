End-to-end credit scoring pipeline: raw bureau data → EDA → feature engineering → model training → threshold optimization → SHAP explainability → deployable scorecard.

## The Problem

A bank needs to decide whether to approve or reject a loan. Approving a good borrower earns $100, but approving someone who defaults costs $500. The goal is to build a model that catches defaults **without** rejecting too many good customers.

The dataset has 3,000 loan applications with 28 credit bureau features (delinquency counts, utilization rates, inquiry history, etc.) and a binary target — good or bad.

### What was wrong with the original model

The original analysis used a single logistic regression and reported "82.83% accuracy." Sounds decent until you realize:

- 83% of loans in the dataset are good. Predicting **everyone** as good gives 83.3% accuracy.
- The model caught only **15 out of 102** bad loans in the test set. That's 85% of defaults slipping through.
- With $500 per missed default, that's a lot of money lost.

The core issues: no handling of class imbalance, no cross-validation, data leakage (imputation before split), and accuracy as the metric on skewed data.

## What I did differently

| Issue | Original | This project |
|-------|----------|-------------|
| Imbalance (83/17) | Ignored | SMOTE + class weights + cost-sensitive |
| Models | Default LogReg | LogReg, Random Forest, XGBoost, LightGBM |
| Validation | One train/test split | 5-fold stratified CV |
| Metric | Accuracy | ROC-AUC, KS statistic, PR-AUC |
| Threshold | Fixed at 0.5 | Optimized for profit ($100/$500 structure) |
| Explainability | None | SHAP values + logistic regression coefficients |
| Deliverable | CSV file | Points-based credit scorecard |

## Key techniques

- **WoE/IV** — the industry standard for credit feature transformation. Bins features, calculates log-odds per bin, ranks predictive power.
- **KS Statistic** — measures max separation between good/bad score distributions. The metric credit risk teams actually use.
- **Profit-maximizing threshold** — instead of the arbitrary 0.5 cutoff, find the threshold that maximizes expected profit given the asymmetric cost structure.
- **SHAP** — explains individual predictions. Important for regulatory compliance (adverse action reasons).
- **Points scorecard** — converts the logistic regression into an additive score. Can be implemented in Excel. Auditors love it.
